产品演示页

产品演示页面要求，刚好一屏不出现滚动条，按指引点击显示内容。
接下来会从这几个点展开讨化。

  • 刚好一屏的实现原理。
  • 项目中遇到的问题。
  • 可扩展方向。

一屏的实现

开始动手前先确定2个问题。

  1. 如何进行“页面切换”或者说是“图片的切换”?
    这种“单页面”(并不是真的单页面应用)让块的隐藏和显示来实现“页面切换”。

  2. 每个用户的浏览器宽高比是不确定的,如何完美的缩放图片比例?
    设计稿采用1920*1080的大小，16：9的比例，获取用户浏览器的宽高，宽度比(图片宽度 / 屏幕宽度)和高度比(图片高度 / 屏幕高度)来确定图片应该以宽度或高度来缩放图片。

遇到的坑

  1. 图片上的marker(可点击区域)等比例缩放(scale)默认参照中心点去缩放，会造成定位对不准，应将参照点设为transform-origin:top left
  2. 图片上的marker(可点击区域)如果内联元素也是绝对定位，自身与内联元素没有文本或背景等设置，在IE11以下无法触发事件。
    bug条件：
    – IE浏览器。
    – 空白元素(没有内联元素，没有设置背景)定位与图片重叠， 重桑部分没法定焦触发事件。

解决方案：
– background: rgba(0,0,0,0); // 推荐，扩展性比较好。
– background: …; opacity: 0; // 如果该元素内还有其它元素，会隐藏子元素。 此项目中刚好有小手动画元素。
– background: url(“#”); // 不推荐，浏览器会去请求是否有此图片。
以上方法都是从背景去考虑，还可以从内联元素去解决。

可扩展方向

目前此版本兼容IE9及以上，谷歌等主流浏览器。主要是因为IE9以下scale不兼容。其实还有一个IE支持的属性zoom可缩放元素，不过zoom最大的坑在于缩放时依然受限于最小12像素中文大小限制，scale不受此限制。如果非要兼容IE7，8，就只能采用zoom方法了。

项目中视频使用的是video标签，不支持IE9以下版本，另外各个浏览器原生的控制按钮宽高大小不一样，有时需要添加一个额外的播放按钮来控制，目前采用的方法是新建等高等宽透明层（大的播放按钮）覆盖在视频上，top: 0;bottom: 60px(没办法获取控制栏的高度);
可使用video.js等插件，它提供了可配置项。

真的单页面应用SPA，可使用Backbone，Knockout，AngularJS，Avalon，VUE等开发框架来制作单页面应用， 以上开发框架都是我复制的，哈哈，有时间找些资料来试一下。
