IOS Input Disabled默认样式问题

IOS 移动端设置了readonly后仍然能被聚焦，后更改为disabled

扩展 - 浏览器输入框记住账号密码后的背景色处理

谷歌浏览器、在记住密码后，通常会给用户密码输入框渲染上一个背景色（rgb(250, 255, 189)）
解决方案原理： 运用box-shadow，inset 设置覆盖默认的背景色。
-webkit-box-shadow: 0 0 0 1000px #fff inset;

box-shadow可覆盖背景色。
-webkit-text-fill-color: #ededed; 可改变字体颜色，后面扩展有提到。

扩展 - 关于-webkit-text-fill-color

表示文字颜色填充，实现的效果基本上与color一样。同时设置-webkit-text-fill-color和color，支持-webkit-text-fill-color浏览器会忽略color值。

扩展 - -webkit-text-stroke

-webkit-text-stroke:1px #000; -webkit-text-stroke第一个参数不为0，第二个参数为空，应用color颜色镂空字体。
-webkit-text-stroke第一个参数不会0，第二个参数不为空（颜色值），应用定义颜色值镂空字体。

这个属性可用于镂空小图标颜色随文本颜色改变而改变（了解一下，毕竟现在可以用iconfont等字体图标）
