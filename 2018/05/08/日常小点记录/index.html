video兼容性处理

优酷视频在IE7以上(IE11的仿真模式下测试)及各个主流浏览器都能正常播放。
360极速模式（极速模式用的是谷歌内核，谷歌浏览器支持，所以很疑惑）和 360极速浏览器不支持。错误原因显示客户端无权播放 201.
百度查找答案显示为： flash插件的原因， 本地测试时更新了flash还是不能播放。

所以把视频传到优酷上是解决视频兼容性的方法之一。

格式 IE Firefox Opera Chrome Safari
Ogg No 3.5+ 5.0+ No No
MPEG 4 9.0+ No No 5.0+ 3.0+
WebM No 4.0+ 10.6+ 6.0+ No

Ogg = 带有 Theora 视频编码和 Vorbis 音频编码的 Ogg 文件
MPEG4 = 带有 H.264 视频编码和 AAC 音频编码的 MPEG 4 文件
WebM = 带有 VP8 视频编码和 Vorbis 音频编码的 WebM 文件

视频编码很重要，如果编码不正确，有的浏览器会渲染为只有声音没有影像。

综上所述： 只要提供ogg和mp4格式就能兼容IE9以上的主流浏览器。

为了兼容IE8及以下浏览器的处理方法：

  • 准备flash文件

  • 使用html5media.js（bootcdn上查找）,让不支持video的浏览器支持video。

  • videojs,等插件库解决方案。（todo 未找到IE8使用flash的问题）

IE,空白块绝对定位元素与图片重叠后点击失效BUG

bug条件：

  1. IE浏览器。
  2. 空白元素(没有内联元素，没有设置背景)定位与图片重叠， 重叠部分无法触发定焦事件。

解决方案：

  1. background: rgba(0,0,0,0); // 推荐，扩展性比较好。
  2. background: …; opacity: 0; // 如果该元素内还有其它元素，会隐藏子元素。
  3. background: url(“#”); // 不推荐，浏览器会去请求是否有此图片。
    以上方法都是从背景去考虑，还可以从内联元素去解决。

查找资料：大多数情况下事件处理函数先执行，但也有例外，比如checkbox的事件默认行为就会先于事件处理函数执行，如果事件处理函数中阻止了默认行为，则会恢复到执行默认行为之前的状态。w3c规范地址

人工理票-模版下拉功能优化

功能说明： 文本输入框初始化为下拉列表(autoSearch.js)，回车后跳转到下一个票据，同时下拉列表为打开状态;
问题： 文本输入框用keyup绑定的事件逻辑为关闭下拉状态，功能和逻辑冲突，当前页面给文本输入框绑定了keydown事件处理票据跳转，事件执行顺序为: keydown，keydown中的focus,插件的keyup。
解决方案： 将自定义事件绑定类型修改为keyup，确保插件的keyup比自定义事件先执行。事件执行顺序为：插件的keyup，自定义keyup，自定义keyup的focus。

总结： 加上断点无法重现bug(无法断点逐行执行)，最后加上多条console.log(“xx”)定位到最后在那调用了Hide()方法。bug定位：花费了很长时间。谨记！
测试了”强制回流”不是bug的原因。

导出文件出错反馈功能

问题：导出文件出错没办法反馈错识信息。
国内云会计导出出错，后台将错误信息反馈到error.jsp,强制用户需要重新登录。这样处理太粗暴。
解决方案： 导出文件前，先请求判断文件是否存在，如果不存在，反馈错识信息，不进行文件导出。先验证，验证成功才导出（这种思想挺重要）

jqGrid单页排序

jqGrid默认排序功能是传递当前列名和排序规则参数，重新发送请求，由后台提供排序后的数据。由于数据量大，请求时间长，项目要求在当前页排序。
功能冲突，并没有重写插件，保留原有方法，在onSortCol回调中返回”stop”，阻止发送远程请求。在方法内获取当前数据进行排序。
数据出现空值，汉字，数字和特殊的值(hour:minute:second)，排序规则：
升序：空值 < 汉字 < 数字
降序：汉字 < 数字 < 空值

排序研究
