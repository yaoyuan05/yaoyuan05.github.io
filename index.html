video兼容性处理

优酷视频在IE7以上(IE11的仿真模式下测试)及各个主流浏览器都能正常播放。
360极速模式（极速模式用的是谷歌内核，谷歌浏览器支持，所以很疑惑）和 360极速浏览器不支持。错误原因显示客户端无权播放 201.
百度查找答案显示为： flash插件的原因， 本地测试时更新了flash还是不能播放。

所以把视频传到优酷上是解决视频兼容性的方法之一。

格式 IE Firefox Opera Chrome Safari
Ogg No 3.5+ 5.0+ No No
MPEG 4 9.0+ No No 5.0+ 3.0+
WebM No 4.0+ 10.6+ 6.0+ No

Ogg = 带有 Theora 视频编码和 Vorbis 音频编码的 Ogg 文件
MPEG4 = 带有 H.264 视频编码和 AAC 音频编码的 MPEG 4 文件
WebM = 带有 VP8 视频编码和 Vorbis 音频编码的 WebM 文件

视频编码很重要，如果编码不正确，有的浏览器会渲染为只有声音没有影像。

综上所述： 只要提供ogg和mp4格式就能兼容IE9以上的主流浏览器。

为了兼容IE8及以下浏览器的处理方法：

  • 准备flash文件

  • 使用html5media.js（bootcdn上查找）,让不支持video的浏览器支持video。

  • videojs,等插件库解决方案。（todo 未找到IE8使用flash的问题）

jqGrid单页排序

jqGrid默认排序功能是传递当前列名和排序规则参数，重新发送请求，由后台提供排序后的数据。由于数据量大，请求时间长，项目要求在当前页排序。
功能冲突，并没有重写插件，保留原有方法，在onSortCol回调中返回”stop”，阻止发送远程请求。在方法内获取当前数据进行排序。
数据出现空值，汉字，数字和特殊的值(hour:minute:second)，排序规则：
升序：空值 < 汉字 < 数字
降序：汉字 < 数字 < 空值

2.1 <script> 元素

带有src属性的 <script>不应该在标签之间再包含额外的JavaScript代码，嵌入的代码会被忽略

<script>标签打开defer或async属性，脚本就会异步加载。渲染引擎遇到这一行命令，就会开始下载外部脚本，但不会等它下载和执行，而是直接执行后面的命令。

defer与async的区别是：前者要等到整个页面正常渲染结束，才会执行；后者一旦下载完，渲染引擎就会中断渲染，执行这个脚本以后，再继续渲染。一句话，defer是“渲染完再执行”，async是“下载完就执行”。另外，如果有多个defer脚本，会按照它们在页面出现的顺序加载，而多个async脚本是不能保证加载顺序的。

数字与字符串比较转换规则
ES5 规范11.9.3.4-5 这样定义：
(1) 如果Type(x) 是数字，Type(y) 是字符串，则返回x == ToNumber(y) 的结果。
(2) 如果Type(x) 是字符串，Type(y) 是数字，则返回ToNumber(x) == y 的结果。

建议不要使用 == true 和 == false。（Boolean类型会首先转换为数字）

对象和非对象之间的相等比较
ES5 规范11.9.3.8-9 做如下规定：
(1) 如果Type(x) 是字符串或数字，Type(y) 是对象，则返回x == ToPrimitive(y) 的结果；
(2) 如果Type(x) 是对象，Type(y) 是字符串或数字，则返回ToPromitive(x) == y 的结果。

这里只提到了字符串和数字，没有布尔值。原因是我们之前介绍过11.9.3.6-7中规定了布尔值会先被强制类型转换为数字。

安全运用隐式强制类型转换

  • 如果两边的值中有true 或者false，千万不要使用==。
  • 如果两边的值中有[]、”” 或者0，尽量不要使用==。

同步与异步的概念

一般而言，操作分为：发出调用和得到结果两步。发出调用，立即得到结果是为同步。发出调用，但无法立即得到结果，需要额外的操作才能得到预期的结果是为异步。同步就是调用之后一直等待，直到返回结果。异步则是调用之后，不能直接拿到结果，通过一系列的手段才最终拿到结果（调用之后，拿到结果中间的时间可以介入其他任务）。
原理：【定时器并不是特例。到达时间点后，会形成一个事件（timeout事件）。不同的是一般事件是靠底层系统或者线程池之类的产生事件，但定时器事件是靠事件循环不停检查系统时间来判定是否到达时间点来产生事件】
原理：【ajax请求是在主线程里面执行，浏览器就会设置侦听来自网络的响应，ajax请求完成事件（异步）放入Event queue】
